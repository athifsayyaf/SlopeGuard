SlopeGuard AI User Manual

Landslide intelligence from terrain, satellite, InSAR, and ML evidence.

1. Opening The Software

Double-click SlopeGuard_AI.exe. The executable starts the local Python backend and opens SlopeGuard AI in a standalone app-style window.

Keep the supporting files beside the EXE: SlopeGuard_AI.dll, SlopeGuard_AI.deps.json, and SlopeGuard_AI.runtimeconfig.json.

2. Interface Layout

  • Left sidebar: Project, DEM, Sentinel-2, InSAR, Inventory, Stack, ML, Interpretation, Report.
  • Top bar: active workflow, layer controls, TIFF/PNG/JPG preview opener.
  • Center: map and raster preview canvas.
  • Right panel: parameters for the selected tool.
  • Bottom panel: status, progress, details, logs, and output files.

3. Typical Workflow

  1. Set project CRS, resolution, AOI, and output folder.
  2. Load DEM and create slope, aspect, TWI, and hillshade.
  3. Create Sentinel-2 indices such as NDVI, EVI, BSI, or NDWI.
  4. Register InSAR velocity and coherence rasters.
  5. Rasterize the landslide inventory mask.
  6. Build the feature stack.
  7. Run ML comparison without InSAR and with InSAR.
  8. Generate interpretation and report outputs.

4. Outputs

Scientific rasters are saved as GeoTIFF. Browser-friendly previews are saved as PNG/JPG.

slope.tif
slope_map.png
slope_map.jpg
model_comparison.csv
slopeguard_report.pdf

5. Viewing TIFF Files

Paste a TIFF path into the top preview box and click Open. SlopeGuard AI creates PNG/JPG previews and displays the map in the canvas.

6. Troubleshooting

Connection refused

The backend did not start. Run this in PowerShell:

cd "E:\Lenovo_path_moved\moved_due_to_storageissue\New project\slideagent-ai"
python backend\server.py

Missing Python package

python -m pip install -r requirements.txt

Permission denied for .ovr

Windows has locked an existing raster overview. The app now saves numbered outputs such as slope_01.tif.
